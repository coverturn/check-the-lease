## STEP 5 — Run Lighthouse and fix anything below 90

**Critical priority — apply full context window and deepest reasoning. Use full autonomy and higher power usage.**

After Steps 1-4 are deployed, run Lighthouse on the homepage in incognito mode (Chrome DevTools → Lighthouse → Performance + Accessibility + Best Practices + SEO + Mobile). Target: **90+ on all four scores.**

### Most likely failures + surgical fixes

**Performance below 90:**
- Add `loading="lazy"` to any image-like SVG that's below the fold
- Defer the chatbot script: don't load the chatbot bundle until the user scrolls past 50% of the viewport OR clicks a "Chat" trigger
- Add explicit `width` and `height` attributes to all illustration SVGs to prevent CLS
- Audit hero illustrations — if any are SVGs being rendered at sizes much smaller than their viewBox, the rasterisation cost is higher than needed
- Wrap the new state-coverage marquee animation in `will-change: transform` (allows GPU acceleration)
- Check for any `@keyframes` animations running on every paint — pause them when offscreen

**Accessibility below 95:**
- Verify `<html lang="en">` is set
- Verify every interactive element has a visible focus ring (sage outline at minimum)
- Audit colour contrast on the new big `50 + DC + 6` block — ink on cream should pass AAA easily
- Floating decorative SVGs all need `aria-hidden="true"`
- Marquee text scroll needs `prefers-reduced-motion` support (already specced in Step 4 — verify)
- Skip-to-main-content link present and functional

**Best Practices below 95:**
- Open the console — no errors, no warnings
- All resources loaded over HTTPS (auto on Replit deploy)
- No deprecated APIs in use
- No browser-storage usage if running in a strict environment

**SEO below 90:**
- `<title>` is unique and descriptive (it is)
- Meta description present (it is)
- Robots meta tag set to `index, follow` (NOT `noindex` — check this!)
- All links have descriptive text (no "click here" anchors)
- All images have meaningful alt text
- Add a sitemap.xml at the root if missing (just listing /, /upload, /how-it-works, /resources, /about, /build-log, /privacy, /terms is enough)

### Run order
1. Deploy Steps 1-4 first
2. Open the live URL in Chrome incognito
3. Open DevTools → Lighthouse tab
4. Select Performance + Accessibility + Best Practices + SEO + Mobile
5. Click "Generate report"
6. Identify any score below 90 and fix the specific items it flags
7. Re-run Lighthouse to confirm
8. Repeat until all four scores are 90+

- [ ] Lighthouse Performance ≥ 90
