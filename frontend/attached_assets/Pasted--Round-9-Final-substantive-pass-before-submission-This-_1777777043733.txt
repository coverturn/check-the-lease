### 6. Add a hero "demo in motion" — CSS-only animation

To replace what would normally be a video, animate the hero's right-side product preview. The flow:

1. Page loads. The dark "FROM YOUR LEASE" card is visible with the lease quote.
2. After 1.5 seconds, a small `↓` arrow animates between the cards (drift down + fade).
3. Then the wine "HIGH RISK" card slides up from below with `spring-pop` easing.
4. Loop with a subtle 4-second pause between cycles, so it feels alive but not nervous.

```css
@keyframes ctl-cycle-arrow {
  0%, 30%, 100% { opacity: 0; transform: translateY(-4px); }
  40%, 70%      { opacity: 1; transform: translateY(0); }
}
@keyframes ctl-cycle-result {
  0%, 35%       { opacity: 0; transform: translateY(20px) scale(0.9); }
  50%, 95%      { opacity: 1; transform: translateY(0) scale(1); }
  100%          { opacity: 0; transform: translateY(20px) scale(0.9); }
}

.hero-arrow {
  animation: ctl-cycle-arrow 8s ease-in-out infinite;
}
.hero-result-card {
  animation: ctl-cycle-result 8s cubic-bezier(0.34, 1.56, 0.64, 1) infinite;
}
```

Wrap in `@media (prefers-reduced-motion: no-preference)`. Reduced motion = static state where both cards are visible always.

This is the "show the product working" move. A real autoplay demo without needing a video file.
