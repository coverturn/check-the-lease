## What to RESTORE (the Wispr design DNA)

### 1. Background colour

Hero and all sections must use **cream `#FBF8F1`** (Lumen) as the page background. Section variation uses `#F2EDE2` (Lumen Dark) for alternating bands.

NO gradients on backgrounds. NO pink-to-blue washes. NO mesh. Just flat cream surfaces.

The dark sections (Problem, final CTA, Founder note's accent block) use **ink-blue `#1E3A5F`** with cream text. Never a gradient.

### 2. Severity colour palette — MUTE IT

The current "HIGH RISK" red card is too aggressive. Replace with the Wispr-calmer palette:

- **High severity:** wine `#7A2C3D` background with cream text (NOT bright red)
- **Medium severity:** clay `#C97A4A` background with cream text
- **Low severity:** sage `#5A8B7A` background with cream text

These are MUTED tones. The energy is "things to watch for" not "DANGER ALERT." Editorial, not panicked.

The hero demo block right side — the small severity badge on the response card — should use these muted tones, not red.

### 3. Restore the marquee curving italic SVG textPath divider

Below the hero, before the Problem section, render the marquee:

```tsx
<svg viewBox="0 0 1200 80" width="100%" height="80">
  <path id="marquee-path" d="M 0,40 Q 300,5 600,35 T 1200,32" fill="none"/>
  <text style={{
    fontFamily: "'Fraunces', Georgia, serif",
```

Optional: animate `startOffset` from 0% to 100% over 30s linear infinite for slow scroll. Disable if `prefers-reduced-motion: reduce`.

### 5. Remove all gradients and shadows

Audit every component. Remove:

- Background gradients (replace with flat cream)
- `box-shadow` on cards (replace with `border: 1.5px solid rgba(23,23,23,0.1)`)
- Glassmorphism or backdrop-filter
- Any drop shadow

The Wispr design DNA: **definition comes from thick borders + huge radius + flat surfaces. Never shadows.**

Card spec to enforce everywhere:
```css
background: #FBF8F1; /* or #F2EDE2 for variation */
border: 1.5px solid rgba(23,23,23,0.1);
border-radius: 16px; /* small cards */ or 24px /* large sections */
box-shadow: none;
transition: transform 0.2s, border-color 0.2s;
```

Hover state:
```css
transform: scale(0.99);
border-color: rgba(23,23,23,0.25);
```

### 6. Restore press-in button hover (NOT scale up)

Audit every button. Replace `:hover { transform: scale(1.05) }` (lift-up) with:

```css
:hover { 
  background: #15294a; /* darker variant */
  transform: scale(0.97); /* press-in */
}
transition: transform 0.15s, background 0.2s;
```

Press-in is the Wispr signature. Lifting up reads as generic SaaS.

### 7. Typography enforcement

Verify all headings use **Fraunces 500** with `font-optical-sizing: auto`. All body uses **Inter 400/500**. All citations use **JetBrains Mono 400**.

Italic Fraunces is reserved for editorial moments only:
- The italic word inside H1 (e.g., the "in plain" in "Your lease, in plain English.")
- The marquee text
- Pull quotes
- Captions

If anywhere is using a non-Fraunces font for headings, fix it. If anywhere is using bold instead of italic for the editorial moments, fix that too.

### 8. Kill the bright RED — soften to wine

Find every instance of bright red (`#FF0000`, `red`, Tailwind `red-500`, etc.) and replace with:

- Wine `#7A2C3D` for "high severity" backgrounds
- Wine-on-cream text for "high severity" labels (cream `#FBF8F1` text on wine `#7A2C3D` background)

The product is calm. Even when flagging dangerous clauses, the visual stays editorial.
