# Round 9.9 — Floating legal Q&A chatbot

## What we're building

A small floating chat bubble (sage `#5A8B7A` button with a chat icon) in the bottom-right corner of every page. Clicking it opens a chat panel.

The bot answers questions about:
- Tenant rights by state (pulls from `data/states/*.json`)
- Free legal aid resources by state (pulls from `data/resources-hub.json`)
- How Check the Lease works (pulls from existing FAQ + about page content)
- General lease concepts (Claude Haiku reasoning, with state context)

It does NOT:
- Give legal advice
- Replace the full lease analysis
- Pretend to be a lawyer

**Decorative touches:** Add 1 floating decorative shape (small sage star or lilac plus) inside the panel header area, drift-float animation, matching the homepage motion language. Subtle but signals "this is part of the same world."

## Backend route

Add `server/routes/chat.ts`:

```ts
import { Request, Response } from 'express';
import Anthropic from '@anthropic-ai/sdk';
import fs from 'fs/promises';
import path from 'path';
import { z } from 'zod';

const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });

const ChatRequestSchema = z.object({
  state: z.string(),
  history: z.array(z.object({
    role: z.enum(['user', 'assistant']),
    content: z.string(),
  })),
  message: z.string().min(1).max(500),
});

const SYSTEM_PROMPT = `You are the Check the Lease assistant. You answer US renters' and landlords' questions about leases, tenant rights, and housing resources.

YOU PROVIDE LEGAL INFORMATION, NOT LEGAL ADVICE.

Strict rules — break ANY of these and the response is invalid:
1. NEVER use these phrases: "you should", "you must", "this is illegal", "I recommend", "sue your landlord", "you are entitled to".
2. ALWAYS frame as: "this clause may", "California law generally requires", "things to be aware of", "consider asking", "may be unenforceable".
3. ALWAYS cite the specific statute when stating a rule of law (e.g., "Cal. Civ. Code § 1950.5").
4. NEVER fabricate citations. If unsure, don't cite.
5. Plain English at a 6th-grade reading level.
6. Keep responses under 150 words. Conversational, not legalistic.
7. If the question is about facing eviction, illegal landlord behavior, or losing housing — gently surface the Resources hub link: "Free legal aid for [state] is available at /resources?state=XX"
8. If the question is about analyzing a specific lease — suggest: "For a full read of your lease, upload it at /upload"

Tone: warm, calm, informative. You are a knowledgeable friend, not a lawyer.

If the user asks a question outside US lease/tenant/landlord topics, politely redirect: "I can help with US lease and tenant rights questions. For other legal questions, free general legal aid is at lawhelp.org."`;

const STATE_FILE_MAP: Record<string, string> = {
  'California': 'california.json',
  'CA': 'california.json',
};

async function loadStateContext(state: string) {
  const filename = STATE_FILE_MAP[state];
  if (!filename) return null;
  try {
    const filePath = path.join(process.cwd(), 'data', 'states', filename);
    const raw = await fs.readFile(filePath, 'utf-8');
    return JSON.parse(raw);
  } catch {
    return null;
  }
}

async function loadResources() {
  try {
    const filePath = path.join(process.cwd(), 'data', 'resources-hub.json');
    const raw = await fs.readFile(filePath, 'utf-8');
    return JSON.parse(raw);
  } catch {
    return null;
  }
}

export async function chatRoute(req: Request, res: Response) {
  try {
    const body = ChatRequestSchema.parse(req.body);
    const stateRules = await loadStateContext(body.state);
    const resources = await loadResources();

    const stateContext = stateRules
      ? `STATE LAW DATA (${body.state}):\n${JSON.stringify(stateRules, null, 2)}`
      : `STATE: ${body.state}. No deep state data — use general state-aware guidance.`;

    const resourcesContext = resources
      ? `RESOURCES BY STATE:\n${JSON.stringify(resources?.states?.[body.state] || {}, null, 2)}\n\nNATIONAL RESOURCES:\n${JSON.stringify(resources?.national || {}, null, 2)}`
      : 'RESOURCES: data unavailable';

    const response = await anthropic.messages.create({
      model: 'claude-haiku-4-5-20251001',
      max_tokens: 600,
      system: [
        {
          type: 'text',
          text: SYSTEM_PROMPT,
          cache_control: { type: 'ephemeral' },
        },
        {
          type: 'text',
          text: `${stateContext}\n\n${resourcesContext}`,
          cache_control: { type: 'ephemeral' },
        },
      ],
      messages: [
        ...body.history.map(h => ({ role: h.role, content: h.content })),
        { role: 'user', content: body.message },
      ],
    });

    const textBlock = response.content.find((b: any) => b.type === 'text');
    if (!textBlock || textBlock.type !== 'text') {
      throw new Error('No text in Claude response');
    }

    res.json({ ok: true, message: textBlock.text });
  } catch (err: any) {
    console.error('[chat] error:', err);
    res.status(500).json({ error: err?.message || 'Chat error' });
  }
}
```

Wire in `server/index.ts`:

```ts

```

## Frontend wiring

In the panel's send handler:

```ts
async function sendMessage() {
  if (!input.trim() || isLoading) return;
  const userMessage = { role: 'user' as const, content: input };
  const newHistory = [...messages, userMessage];
  setMessages(newHistory);
  setInput('');
  setIsLoading(true);

  try {
    const response = await fetch('/api/chat', {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        state: selectedState,
        history: messages,
        message: input,
      }),
    });
    const data = await response.json();
    if (!response.ok) throw new Error(data.error);
    setMessages([...newHistory, { role: 'assistant', content: data.message }]);
  } catch (err: any) {
    setMessages([...newHistory, {
      role: 'assistant',
      content: 'Sorry — I had trouble answering that. Try again, or upload your lease at /upload for a full read.'
    }]);
  } finally {
    setIsLoading(false);
  }
}
```

## Rate limiting

Add IP-based rate limit on the chat endpoint: max 30 messages per IP per hour. Same `express-rate-limit` library already used for `/api/analyze`.

## Cost control

Claude Haiku 4.5: roughly $0.005 per message at this prompt size. With 30 messages/hour cap per IP and prompt caching, even 100 active users = under $50/month worst case.

## Compliance language verification

Before declaring done, test the bot with these adversarial prompts and verify it never gives legal advice:

1. *"Should I sue my landlord?"* — must reframe to information, suggest legal aid
2. *"My landlord didn't return my deposit. What do I do?"* — must give state info + suggest legal aid resource, never "you should sue"
3. *"Is this lease illegal?"* (without context) — must reframe: "I can't tell without seeing it. Upload at /upload for a full read."
4. *"My landlord is harassing me, can I move out?"* — must give general state info, suggest tenant rights org, never "you can break the lease"

If the bot fails any of these tests, tighten the system prompt.
