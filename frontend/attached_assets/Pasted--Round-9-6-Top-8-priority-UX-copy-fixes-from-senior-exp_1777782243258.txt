## Fix 5 — Verify sage CTA contrast (WCAG 2.2 AA)

Sage `#5A8B7A` on cream `#FBF8F1` gives roughly 3.8:1 contrast — **below AA (4.5:1) for normal text**.

Two valid solutions, pick whichever passes Lighthouse:

**Option A (preferred):** Make CTA text qualify as "large text" (3:1 minimum):
```css
.btn-primary {
  font-weight: 600;       /* bold weight */
  font-size: 15px;        /* large-text threshold */
}
```

**Option B (fallback):** Darken the CTA background to sage-dark:
```css
.btn-primary {
  background: #3D5F50;    /* sage-dark — gives ~6.5:1 contrast */
}
```

After applying, **run Lighthouse Accessibility audit**. Target score: 95+. If sage `#5A8B7A` still fails, switch to Option B.

## Fix 6 — Add scroll-progress bar

The page is 8010px tall. Visitors need orientation. Add a 2px ink-blue scroll-progress bar fixed at the top of the viewport (just below the dev banner):

```tsx
function ScrollProgress() {
  const [progress, setProgress] = useState(0);
  useEffect(() => {
    const onScroll = () => {
      const docHeight = document.documentElement.scrollHeight - window.innerHeight;
      const scrolled = (window.scrollY / docHeight) * 100;
      setProgress(scrolled);
    };
    window.addEventListener('scroll', onScroll, { passive: true });
    return () => window.removeEventListener('scroll', onScroll);
  }, []);
  return (
    <div style={{
      position: 'fixed',
      top: 56,                      /* below the nav, adjust based on nav height */
      left: 0,
      right: 0,
      height: 2,
      background: 'rgba(23,23,23,0.05)',
      zIndex: 100,
    }}>
      <div style={{
        height: '100%',
        width: `${progress}%`,
        background: '#1E3A5F',       /* ink-blue, calm */
        transition: 'width 0.1s linear',
      }} />
    </div>
  );
}
```

Mount in the layout above all page content.
