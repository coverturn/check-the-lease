## Tasks (in this order)

### 1. Improve the loading state on `/upload`

When the user clicks "Analyse my lease" or "Try the sample California lease" and analysis is in progress, replace the form area with a centred loading card. The current state has no animation and users get confused.

Loading card spec:
- Replaces the form area entirely (don't render the form during analysis)
- Centred horizontally, padding 48px 32px, border-radius 24px
- Background: bone-dark (`#F2EDE2`)
- Border: 1.5px soft (rgba(23,23,23,0.1))
- Inside: a sage dot (12px diameter, `#5A8B7A`) at the top, pulsing
- Below the dot: Fraunces italic 28px ink: rotating message
- Below the message: Inter 13px ink-muted: "California law check · about 5–15 seconds"

Pulse animation (CSS):
```css
@keyframes ctl-pulse {
  0%, 100% { transform: scale(1); opacity: 1; }
  50% { transform: scale(1.6); opacity: 0.4; }
}
.ctl-loading-dot { animation: ctl-pulse 1.5s ease-in-out infinite; }
```

Rotating messages (cycle every 3 seconds while `isAnalysing` is true):
1. "Reading your lease."
2. "Checking California law."
3. "Surfacing what matters."
4. "Almost there."

Use `useEffect` with `setInterval` to rotate the message index. Clear the interval when `isAnalysing` becomes false.

When the user clicks "Try the sample California lease," set `isAnalysing` to true IMMEDIATELY before the fetch starts. Don't wait for the fetch to begin — set it on click, then start the fetch. Even if the cached demo result returns in 1ms, the user should see at least the loading card flash for ~600ms (set a `setTimeout` to delay navigation by 600ms after the fetch resolves, so the loading state is visible).

### 2. Extend the analysis schema to include parent considerations

Update `server/lib/schema.ts` to add a new optional field on `AnalysisResultSchema`:

```ts
export const ParentConsiderationSchema = z.object({
  title: z.string(),
  explanation: z.string(),
});

export const AnalysisResultSchema = z.object({
  key_terms: z.array(KeyTermSchema),
  potential_issues: z.array(PotentialIssueSchema),
  missing_protections: z.object({
    renter: z.array(MissingProtectionSchema),
    landlord: z.array(MissingProtectionSchema),
  }),
  parent_considerations: z.array(ParentConsiderationSchema).optional(),
  questions: z.array(z.string()),
  stats: z.object({
    potential_issues: z.number(),
    missing_protections: z.number(),
    questions: z.number(),
  }),
});
```

### 3. Update the Claude system prompt to surface parent considerations

In `server/lib/runAnalysis.ts`, update the user prompt builder. When `intake.isParent` is true, instruct Claude to also populate a `parent_considerations` array in the response.

Add this conditionally to the user prompt when `intake.isParent` is true:

```
Because the user marked themselves as a parent or guardian, ALSO populate a parent_considerations field in your JSON response. Include 3-5 items relevant to family stability:

- Occupancy/guest limits that may affect children
- Renewal language affecting school stability
- Habitability with children in mind (heating, hot water, mold)
- Retaliation protections if reporting code violations
- Repair urgency for essential services
- Any clauses that may have specific implications for households with children

Format each as: { "title": "Short plain-English title", "explanation": "Why this matters as a parent." }

Use the same compliance language rules — never legal advice, frame as information.
```

When `intake.isParent` is false, do NOT include the parent_considerations array in the response.

Also update the JSON SCHEMA section in the user prompt to include the new field conditionally.

You will also need to invalidate the cached demo result if the schema changes — delete `data/cached-demo-result.json` so the next demo run regenerates it.

### 5. Wire the Resources section on the results page

The results page already has a Resources section shell from Round 3 (placeholder data). Replace with real data from `data/resources/california.json`.

Logic:
- If user is in California (`intake.state === 'California' || intake.state === 'CA'`), import the CA resources file directly into the page (Vite's static asset import, or a small `/api/resources/:state` endpoint — your call, prefer the simpler import).

- Filter logic: a program shows if its `filter_tags` includes the active filter. If the active filter is "all", show all programs.

- Render the existing dark ink-blue panel design from Round 3.
